MiniBank app

Консольное банковское приложение, разработанное на Java с использованием чистого **Spring Core**.

Проект создан с учебной целью для глубокого погружения в принципы работы Spring Framework (Inversion of Control, Dependency Injection, жизненный цикл бинов), без использования магии автоконфигураций Spring Boot. Все настройки контекста и связывание компонентов выполнены вручную через Java-конфигурацию (`@Configuration`, `@Bean`).

## Технологический стек
* **Язык:** Java 17+
* **Фреймворк:** Spring Context (Core) 6.1+
* **Сборка:** Maven
* **Архитектура:** Построена на основе паттерна "Стратегия" (Command Processor Pattern). Для маршрутизации консольных команд используется `Map`, собираемая динамически через Stream API при инициализации Spring-контекста.

## Реализованный функционал
Приложение работает в интерактивном консольном цикле и поддерживает следующие операции:
* `Создать_пользователя` — регистрация нового клиента с проверкой уникальности логина.
* `Создать_аккаунт` — открытие банковского счета. При открытии первого счета пользователю автоматически начисляется приветственный бонус.
* `Пополнение_счета` / `Снятие_средств` — базовые финансовые транзакции с проверкой достаточности баланса и защитой от отрицательных сумм.
* `Перевод_средств` — переводы между счетами. Включает бизнес-логику: переводы между своими счетами выполняются без комиссии, при переводе другому пользователю взимается процент.
* `Закрытие_счета` — закрытие счета с автоматическим переводом остатка на другой активный счет пользователя.
* `Список_пользователей` — просмотр состояния базы данных в памяти.

*Примечание: в данной версии проекта данные хранятся в оперативной памяти (в коллекциях `HashMap` и `HashSet`) и сбрасываются при завершении работы.*

## Настройки приложения
Базовые параметры вынесены в файл `src/main/resources/application.properties` и внедряются в сервисы с помощью аннотации `@Value`:
* `account.default-amount` — размер приветственного бонуса для новых пользователей.
* `account.transfer-commission` — размер комиссии при переводах между разными пользователями (например, 0.02 = 2%).
